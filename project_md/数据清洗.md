# 医疗问答知识库：数据清洗+数据增强 完整落地方案
（全程贴合医疗场景「合规、准确、无推理、可审核」核心原则，无通用场景的风险操作，面试能详细讲、落地能直接执行）

## 核心前置原则（必须先明确，避免踩坑）
1. **清洗核心**：去噪、标准化、合规化，只保留「权威、完整、无违规」的医疗科普信息，剔除所有诊断/治疗/用药等红线内容；
2. **增强核心**：只做「语言层、非推理式」增强，不新增任何医学信息、不改变原意，仅提升检索覆盖度；
3. **全程约束**：所有操作可追溯、可审核，医疗内容必须经医学专业人员校验，杜绝幻觉/错误信息。

---

## 一、数据清洗：医疗场景专属7步流程（从粗到细，层层过滤）
数据清洗是知识库的“地基”——医疗文档（如卫健委指南、三甲医院科普）常包含格式冗余、无关内容、甚至隐性违规表述，必须逐轮清洗才能入库。

### 步骤1：基础格式清洗（通用但医疗需适配）
**核心目标**：消除格式干扰，让文档结构化，便于后续处理

| 操作内容 | 具体示例 | 医疗场景注意事项 |
| --- | --- | --- |
| 剔除冗余格式 | 去掉PDF/Word中的水印、页眉页脚、广告栏、多余换行/空格 | 医疗文档常带“XX医院宣传语”，需完全剔除，只留科普内容 |
| 统一文本编码 | 将GBK/乱码文档转为UTF-8，修复乱码字符 | 避免医疗术语（如“转氨酶”）因乱码变成错误表述 |
| 拆分长文档 | 按语义块拆分超长篇文档（如《孕产期保健指南》拆分为“孕早期护理”“孕晚期宫缩”等子文档） | 拆分后需保证每个子文档语义完整，不割裂医疗知识 |
| 统一文本格式 | 将表格/图片中的医疗信息转为纯文本（如“产检项目表”→ 文字版“孕12周产检项目：NT检查、血常规”） | 表格中的“正常范围”“注意事项”需完整提取，不遗漏核心数据 |


### 步骤2：文本冗余清洗（医疗专属，去无关内容）
**核心目标**：只保留医疗科普核心，剔除所有非必要内容

| 操作内容 | 具体示例 | 目的 |
| --- | --- | --- |
| 剔除无关推广/案例 | 删掉“XX医院产科是重点科室，专家团队强大”“某患者经我院治疗后康复”等非科普内容 | 避免检索时匹配到无关信息，干扰精准度 |
| 删除语气/情绪化表述 | 删掉“千万别忽视！”“超实用！”“我建议大家”等主观语气词 | 医疗科普需客观，剔除情绪化内容保证严谨性 |
| 合并重复段落 | 同一文档中“孕晚期胎动正常范围”重复出现2次 → 保留1版完整内容 | 避免知识库冗余，降低检索成本 |


### 步骤3：医疗术语标准化清洗（核心，和Query改写对齐）
**核心目标**：让文档术语与用户Query改写后的关键词统一，提升检索精准度

| 操作内容 | 具体示例 | 落地依据 |
| --- | --- | --- |
| 口语/俗称→标准医学术语 | “肚子硬邦邦的”→“假性宫缩”；“嗓子疼”→“咽痛”；“血压高”→“高血压” | 基于你之前的《医疗口语-标准词映射表》，统一术语 |
| 修正错别字/简写 | “糖鸟病”→“糖尿病”；“孕产捡”→“产前检查”；“甲低”→“甲状腺功能减退” | 基于《医疗错词-正确词映射表》，避免术语错误 |
| 统一术语表述 | 把“孕检”“产检”统一为“产前检查”；“胎动次数”“胎动频率”统一为“胎动正常范围” | 避免同一概念多种表述导致检索漏匹配 |


### 步骤4：合规性清洗（医疗红线，必须100%执行）
**核心目标**：剔除所有违规内容，守住医疗合规底线（这是医疗清洗最关键的一步）

| 操作内容 | 需剔除的违规内容示例 | 替代处理方式 |
| --- | --- | --- |
| 剔除诊断/治疗建议 | “胎动少就是胎儿缺氧”“头疼可吃布洛芬缓解”“胃炎需服用奥美拉唑” | 仅保留科普：“胎动少需及时就医”“头疼可通过休息缓解，具体需遵医嘱” |
| 剔除用药剂量/方案 | “高血压患者每日吃1片硝苯地平”“孕期补充叶酸0.8mg/天” | 改为：“高血压患者用药需遵医嘱”“孕期叶酸补充需咨询医生” |
| 剔除急症处理细节 | “心梗发作需立即吃硝酸甘油”“大出血需按压XX部位” | 改为：“心梗发作需立即拨打120”“大出血需及时就医” |
| 过滤敏感/隐私内容 | 病历号、处方号、患者姓名、具体医院科室电话 | 完全删除，不保留任何隐私信息 |
| 剔除模糊/不确定表述 | “可能是XX病”“大概率是早产”“建议做XX检查” | 改为客观表述：“出现XX症状需及时就医检查” |


### 步骤5：权威校验清洗（医疗专属，保证内容准确）
**核心目标**：只保留权威来源内容，杜绝错误科普

| 操作内容 | 具体执行 | 淘汰标准 |
| --- | --- | --- |
| 核验内容来源 | 仅保留“国家卫健委/国家疾控中心/三甲医院官网/药典/医学教材”来源的内容 | 淘汰自媒体、养生公众号、非权威机构的科普内容 |
| 修正错误医疗信息 | 如“孕期可多吃山楂”（核实山楂易引发宫缩）→ 直接删除该段落 | 错误内容不修正，直接淘汰，避免误导 |
| 核对核心数据 | 如“胎动正常范围”“血压正常值”→ 核对最新卫健委指南，修正过时数据 | 淘汰2015年前的老旧内容（医疗指南常更新） |


### 步骤6：语义完整性清洗（医疗专属，避免碎片化）
**核心目标**：保证每个清洗后的文本块是「完整的科普单元」，避免检索到碎片化信息

| 操作内容 | 具体要求 | 示例 |
| --- | --- | --- |
| 检查核心三要素 | 每个文本块必须包含：①定义/表现 ②注意事项 ③就医提示（缺一项则处理） | 原内容“胎动少需就医”→ 补充为“胎动少是孕晚期常见现象，正常胎动次数为每小时3-5次，若胎动少于该范围需及时就医” |
| 剔除碎片化文本 | 仅保留“假性宫缩”“转氨酶高”等单术语，无任何解释 → 直接淘汰 | 避免检索到无意义的单术语，导致回答空洞 |
| 补充缺失信息 | 从同权威来源补充缺项内容（如缺“就医提示”→ 从卫健委指南中截取） | 保证内容完整，不遗漏关键就医提示（医疗场景核心） |


### 步骤7：质量校验（最后一道关，必须人工审核）
| 操作内容 | 执行标准 | 目的 |
| --- | --- | --- |
| 医学人员抽检 | 抽检比例≥20%，重点检查合规性、准确性 | 机器无法识别隐性医疗错误，必须人工把关 |
| 自动关键词校验 | 用规则检查是否含违规词（如“诊断”“治疗”“用药剂量”） | 拦截漏网的违规内容 |
| 人工通读核心段落 | 通读“孕期护理”“慢病饮食”等高频内容，确保表述严谨 | 避免“口语化过度”“表述模糊”等问题 |

## 二、数据增强：医疗场景仅能做的5种「轻量、非推理式」增强
医疗场景**绝对禁止**用大模型生成式增强（易编造假医疗信息），仅能做「语言层、不新增医学信息」的增强，核心目的是提升检索覆盖度（适配用户不同的提问方式）。

### 增强方法1：医疗术语同义词替换（核心）
**核心逻辑**：在不改变原意的前提下，替换为同义标准术语，生成多版本文本，适配用户不同的术语表述

| 操作示例 | 注意事项 |
| --- | --- |
| 原文本：“孕晚期假性宫缩”→ 版本1：“妊娠晚期假性宫缩”；版本2：“孕晚期生理性宫缩” | ① 仅替换术语，不改变语义；② 替换后需医学审核，确保术语同义；③ 每个文本最多生成2个版本，避免冗余 |


### 增强方法2：句式规整增强
**核心逻辑**：将口语化/倒装句式改为标准陈述句，生成多版本，适配用户不同的提问句式

| 操作示例 | 注意事项 |
| --- | --- |
| 原文本：“孕晚期多少胎动正常？”→ 版本1：“孕晚期正常胎动次数是多少”；版本2：“妊娠晚期胎动正常范围” | ① 只改句式，不增删医疗信息；② 避免生成疑问句（知识库存陈述句更适配检索） |


### 增强方法3：场景补全增强（医疗专属）
**核心逻辑**：为通用内容补充「权威明确的人群/场景标签」，生成细分版本，提升精准检索

| 操作示例 | 注意事项 |
| --- | --- |
| 原文本：“高血压饮食需低盐”→ 版本1：“老年高血压患者饮食需低盐”；版本2：“合并糖尿病的高血压患者饮食需低盐” | ① 补全的场景必须来自权威指南（如《高血压防治指南》）；② 不主观编造场景（如“孕妇高血压”需核实是否有权威依据） |


### 增强方法4：多版本归一化增强（反向增强）
**核心逻辑**：将不同表述的同一内容归一化为「标准版本」，避免检索碎片化

| 操作示例 | 目的 |
| --- | --- |
| 原内容1：“孕晚期宫缩”；原内容2：“妊娠晚期子宫收缩”；原内容3：“孕38周肚子发紧”→ 归一为“孕晚期宫缩（假性宫缩）” | 让用户无论用哪种表述提问，都能匹配到同一标准内容，提升检索效率 |


### 增强方法5：关键词标注增强（辅助检索）
**核心逻辑**：为清洗后的文本块标注「核心关键词」（和向量库标引对齐），生成“文本+关键词”的增强版本

| 操作示例 | 目的 |
| --- | --- |
| 文本：“孕晚期假性宫缩是常见的生理现象，表现为腹部发紧、无规律性，休息后可缓解，若规律宫缩需就医”→ 标注关键词：孕晚期、假性宫缩、生理现象、腹部发紧、规律宫缩 | 辅助向量库检索，让Query向量能快速匹配到核心关键词，提升召回率 |


### 医疗场景「绝对禁止」的增强方式（红线）
1. ❌ 生成式增强：用大模型生成“胎动少的原因”“高血压的治疗方法”等内容（易产生医学幻觉）；
2. ❌ 推理式增强：补充“胎动少可能是缺氧”“头疼是颈椎病引起的”等诊断类内容（越界）；
3. ❌ 扩展式增强：从“低盐饮食”扩展到“低盐饮食的具体菜品”（新增未审核内容，易出错）；
4. ❌ 跨场景增强：从“孕期宫缩”扩展到“分娩宫缩”（改变语义范围，易误导）。

### 关键落地细节

## 四、面试口述精简版（1分钟直接背）
在医疗问答知识库的前期处理中，我会遵循「清洗优先、增强为辅，全程非推理、可审核」的核心原则：

数据清洗分7步落地：先做基础格式清洗消除格式干扰，再剔除冗余内容只留核心科普，接着将口语/错字标准化为医疗术语，然后严格剔除诊断、用药等违规内容，再校验来源权威度、补充完整语义，最后经医学审核做质量把关；

数据增强仅做5种轻量操作：医疗术语同义词替换、句式规整、权威场景补全、多版本归一化、关键词标注，绝对不做生成式、推理式增强，避免引入错误信息或越界内容。

整套流程既保证了知识库内容的权威合规，又提升了后续检索的精准度和覆盖度，完全贴合医疗场景的安全要求。

## 总结
1. 数据清洗核心是「去噪、标准化、合规化、权威化」，剔除所有违规/错误内容，保证文本完整且符合医疗红线；
2. 数据增强仅做「语言层」操作，不新增医学信息、不推理，核心是提升检索覆盖度；
3. 全程需医学审核+操作留痕，确保内容准确、可追溯，这是医疗知识库的核心底线。
