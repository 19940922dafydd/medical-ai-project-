# 🏥 医疗问答智能体产品说明书 (PRD)

## 1. 项目背景与行业痛点 (Market Background)

在医疗大模型落地过程中，行业普遍面临「**最后一公里**」难题：
- **准确性风险**: 通用 LLM 容易产生幻觉，在药理配伍、临床诊断等严肃场景下可能导致误导性建议。
- **知识孤岛**: 医疗文献、院内规章、临床指南广泛分布在 PDF、Word、数据库及专家大脑中，缺乏统一的消费接口。
- **系统黑盒**: 传统的 RAG 系统难以解释检索了哪些内容、为何得出该结论，导致医护人员信任度低。
- **进化停滞**: 无法将用户反馈（点击、负评）自动转化为系统的知识补丁。

**本产品定位**: 构建一个**可解释、可进化、专业级**的医疗知识决策中枢。

## 3. 功能全景图 (Functional Architecture)

### 3.1 智能分流与专家系统 (Smart Routing)
- **多意图拆解**: 将嵌套 Query（如「我想去挂内科，顺便问问感冒药怎么吃」）自动拆解为「行政」与「用药」两个原子任务。
- **动态寻路**: 基于 LangGraph 的条件路由，实现任务在不同专家节点（Admin/Pharma/Diag）间的精准流转。

### 3.2 混合数据库矩阵 (Mixed Storage)
- **语义池 (Vector)**: 百万级切片存储，支持 L1-L3 层级化召回，避免领域知识污染。
- **事实网 (Graph)**: 存储药物成分、禁忌症等「硬逻辑」，为模型生成提供确定性约束。

### 3.3 企业级管理后台
- **知识洗练**: 支持 PDF/Docx/CSV 一键入库，内置文本清洗与三元组自动提取。
- **白盒联调**: 实时追踪 Agent 推理状态，包含 Node 耗时、检索距离 (Distance) 及图谱路径。

## 4. 核心亮点：数据飞轮 (HITL Data Flywheel)

本产品的差异化价值在于其自闭环的反馈系统：

1.  **捕获 (Capture)**: 业务侧通过「Bad Case」按钮记录交互 ID。
2.  **归因 (Analysis)**: 系统自动回溯当次请求的完整链路（包括召回的 Chunk 和提取的图谱节点）。
3.  **修复 (Refine)**:
    - **词典重写**: 将误判的口语词强转为标准词。
    - **知识增补**: 针对检索真空，快速注入补充文献。
4.  **回归 (Regress)**: 一键执行「单例跑测」，确保该 Case 已被修复且未产生副作用（Regressions）。

## 5. 项目指标 (KPIs / Success Metrics)

- **回答准确率 (Accuracy)**: 经由专家审评的 Ground Truth 对比通过率（目标 > 95%）。
- **幻觉拦截率 (Fact-Check Rate)**: `fact_checker` 节点捕获出的不一致回答占比。
- **响应耗时 (Latency)**: 首字生成时间 (TTFT) < 150ms，端到端生成 < 2s。
- **人为干预成本**: 解决单个 Bad Case 的平均时长（从反馈到上线 < 5mins）。
